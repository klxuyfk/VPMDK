# USPEX 9.4.4 Si Example

This directory contains a minimal USPEX 9.4.4 input deck demonstrating how
`vpmdk` can be used as the executable inside a VASP-oriented crystal-structure
search workflow.

Included files:

- `INPUT.txt`: top-level USPEX configuration with `commandExecutable` set to
  `vpmdk > log`
- `Specific/INCAR_1` ... `Specific/INCAR_5`: stage-specific VASP-style control
  files used by USPEX local optimizations

Notes:

- This is an interoperability example, not a standalone benchmark.
- `POTCAR` content is intentionally omitted because VASP pseudopotentials are
  not redistributable here.
- Structures are generated by USPEX during the search, so no fixed `POSCAR` is
  bundled in this example directory.
